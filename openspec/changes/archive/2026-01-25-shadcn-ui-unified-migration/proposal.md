# 统一迁移 shadcn/ui 组件库和配置

## 概述
本提案旨在将当前项目的组件库和配置体系完全迁移到与 `/home/newbe36524/repos/newbe36524/pcode/src/PCode.Client` 相同的 shadcn/ui 组件库和配置标准，以提高组件一致性、可维护性和开发效率。

## 背景
当前项目使用自定义的组件和配置体系，存在以下问题：
- 组件库碎片化，维护成本高
- 配置可能与 shadcn/ui 最佳实践不符
- 缺少统一的组件开发和使用标准
- 组件版本管理混乱

源项目 PCode.Client 使用 shadcn/ui 组件库，具有完整的组件集合、统一的配置体系和最佳实践，值得本项目借鉴和统一。

## 目标
1. 统一组件库：完全复制 PCode.Client 的 shadcn/ui 组件到当前项目
2. 统一配置：同步复制 shadcn/ui 相关的配置文件
3. 统一样式：使用相同的 Tailwind 配置和全局样式
4. 统一开发模式：确保所有组件使用 shadcn/ui 的设计规范

## 范围
- **组件迁移**：复制 PCode.Client 中的所有 shadcn/ui 组件（`src/components/ui/` 目录）
- **配置迁移**：
  - `components.json`：shadcn/ui 配置文件
  - `tailwind.config.js`：Tailwind CSS 配置
  - `src/index.css`：全局样式文件
  - `package.json`：依赖配置

## 影响
- 提高组件一致性和可维护性
- 降低新功能开发成本
- 统一技术栈，提升开发效率
- 可能需要更新相关文档和使用示例
- 现有组件使用代码需要相应调整

## 风险评估
1. **组件兼容性风险**：源项目组件可能使用了当前项目未安装的依赖
2. **样式冲突风险**：不同的 Tailwind 配置和全局样式可能导致布局问题
3. **代码调整风险**：现有组件使用方式可能需要修改以适配新组件 API

## 成功标准
1. 所有 shadcn/ui 组件能够正常导入和使用
2. 项目能够正常构建和运行
3. 组件样式与源项目保持一致
4. 现有功能不受迁移影响

- 源项目：`/home/newbe36524/repos/newbe36524/pcode/src/PCode.Client/`
- 目标项目：当前项目根目录
